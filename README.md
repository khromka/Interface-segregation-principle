# Interface Segregation Principle (ISP) in C++

## Опис
Принцип розділення інтерфейсу (ISP) каже: *"Клієнти не повинні залежати від методів, які вони не використовують."*  
Тому краще створювати кілька вузьких інтерфейсів, ніж один великий.

## Приклад
У цьому репозиторії:
- `IWorkable` — інтерфейс для роботи
- `IFeedable` — інтерфейс для харчування
- `Human` реалізує обидва інтерфейси
- `Robot` реалізує лише `IWorkable`

Таким чином, **Robot не змушений реалізовувати метод eat()**, що відповідає ISP.
